### 58. ¿Qué son los Virtual Threads?

**¿Qué es?** Threads ligeros gestionados por la JVM (no el OS). Puedes crear millones.

**¿Para qué sirve?** Operaciones I/O-bound (HTTP, BD). NO para CPU-intensive.

<Code code={`// Virtual Threads - threads ligeros (Project Loom)

    System.out.println("Virtual thread");

// Con ExecutorService
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 10000).forEach(i -> {
        executor.submit(() -> {
            Thread.sleep(Duration.ofSeconds(1));
            return i;
        });
    });
}

// Diferencias con Platform Threads:
// - Muy ligeros (KB vs MB)

### 59. ¿Qué es un Record?

**¿Qué es?** Clase inmutable para transportar datos. Genera automáticamente constructor, getters, equals, hashCode, toString.

**¿Para qué sirve?** DTOs, value objects, eliminar boilerplate.

<Code code={`// Record - clase inmutable para datos
public record Persona(String nombre, int edad) {
    // Constructor compacto para validación
    public Persona {
        if (edad < 0) {
            throw new IllegalArgumentException("Edad negativa");
        }
        nombre = nombre.trim();
    }
    
    // Métodos adicionales
    public String nombreMayusculas() {
        return nombre.toUpperCase();
    }
    
    // Constructor adicional
    public Persona(String nombre) {
        this(nombre, 0);
    }
}

// Uso
Persona p = new Persona("Juan", 25);
String nombre = p.nombre(); // getter
int edad = p.edad();

// Automáticamente tiene:
// - Constructor con todos los campos
// - Getters (nombre(), edad())
// - equals() y hashCode()
// - toString()`} lang="java" />

### 60. ¿Qué son las Sealed Classes?

**¿Qué es?** Clases que restringen explícitamente qué otras clases pueden heredar de ellas.

**¿Para qué sirve?** Modelar dominios cerrados, mejor pattern matching, control de jerarquía.

<Code code={`// Sealed class - restringe qué clases pueden heredar
public sealed class Figura 
    permits Circulo, Rectangulo, Triangulo {
}

// Subclases deben ser: final, sealed, o non-sealed
public final class Circulo extends Figura {
    private double radio;
}

public sealed class Rectangulo extends Figura 
    permits Cuadrado {
}

public final class Cuadrado extends Rectangulo {}

public non-sealed class Triangulo extends Figura {
    // Cualquiera puede heredar de Triangulo
}

// Útil para pattern matching exhaustivo
double area = switch (figura) {
    case Circulo c -> Math.PI * c.getRadio() * c.getRadio();
    case Rectangulo r -> r.getAncho() * r.getAlto();
    case Triangulo t -> t.getBase() * t.getAltura() / 2;
    // No necesita default - es exhaustivo
};`} lang="java" />

### 61. ¿Qué es Pattern Matching?

**¿Qué es?** Extraer datos de objetos de forma declarativa, combinando instanceof + cast + asignación.

**Tipos:** Para instanceof, para switch, para records (deconstrucción).

<Code code={`// Pattern Matching para instanceof
Object obj = "Hola";

// Antes
if (obj instanceof String) {
    String s = (String) obj;
    System.out.println(s.length());
}

// Ahora
if (obj instanceof String s) {
    System.out.println(s.length());
}

// Pattern Matching en switch
String resultado = switch (obj) {
    case Integer i when i > 0 -> "Positivo: " + i;
    case Integer i -> "No positivo: " + i;
    case String s when s.length() > 5 -> "String largo";
    case String s -> "String corto: " + s;
    case null -> "Nulo";
    default -> "Otro tipo";
};

// Record Patterns
record Point(int x, int y) {}
record Line(Point start, Point end) {}

void procesar(Object obj) {
    if (obj instanceof Line(Point(var x1, var y1), Point(var x2, var y2))) {
        System.out.println("Línea de (" + x1 + "," + y1 + ") a (" + x2 + "," + y2 + ")");
    }
}`} lang="java" />

### 63. ¿Qué son las Switch Expressions?

**¿Qué es?** Switch que retorna un valor. Usa `->` en vez de `:` y no necesita break.

**Ventajas:** Más conciso, sin fall-through accidental, exhaustivo.

<Code code={`// Switch tradicional (statement)
String tipo;
switch (dia) {
    case LUNES:
    case MARTES:
        tipo = "Inicio";
        break;
    default:
        tipo = "Otro";
}

// Switch expression (Java 14+)
String tipo = switch (dia) {
    case LUNES, MARTES -> "Inicio";
    case MIERCOLES -> "Mitad";
    case JUEVES, VIERNES -> "Fin laboral";
    case SABADO, DOMINGO -> "Fin de semana";
};

// Con yield para bloques
String descripcion = switch (dia) {
    case LUNES -> {
        System.out.println("Procesando lunes");
        yield "Primer día";
    }
    default -> "Otro día";
};`} lang="java" />

## Patrones de Diseño (66-80)

### 66. ¿Qué es el patrón Factory?

**¿Qué es?** Patrón que delega la creación de objetos a un método/clase especializada.

**¿Para qué sirve?** Desacoplar creación de uso, crear objetos según condiciones.

<Code code={`// Factory Method
public interface Animal {
    void hablar();
}

public class Perro implements Animal {
    public void hablar() { System.out.println("Guau"); }
}

public class Gato implements Animal {
    public void hablar() { System.out.println("Miau"); }
}

public class AnimalFactory {
    public static Animal crear(String tipo) {
        return switch (tipo.toLowerCase()) {
            case "perro" -> new Perro();
            case "gato" -> new Gato();
            default -> throw new IllegalArgumentException("Tipo desconocido");
        };
    }
}

// Uso
Animal animal = AnimalFactory.crear("perro");`} lang="java" />

### 67. ¿Qué es el patrón Builder?

**¿Para qué sirve?** Objetos con muchos parámetros opcionales, inmutabilidad, legibilidad.

### 68. ¿Qué es el patrón Observer?

**¿Qué es?** Patrón donde un objeto (subject) notifica cambios a múltiples objetos dependientes (observers).

**¿Para qué sirve?** Eventos, notificaciones, actualizaciones en tiempo real.

<Code code={`import java.util.*;

// Subject
public class EventoNotificador {
    private List<Observador> observadores = new ArrayList<>();
    
    public void agregar(Observador o) {
        observadores.add(o);
    }
    
    public void remover(Observador o) {
        observadores.remove(o);
    }
    
    public void notificar(String evento) {
        for (Observador o : observadores) {
            o.actualizar(evento);
        }
    }
}

// Observer
public interface Observador {
    void actualizar(String evento);
}

public class EmailObservador implements Observador {
    @Override
    public void actualizar(String evento) {
        System.out.println("Email: " + evento);
    }
}

// Uso
EventoNotificador notificador = new EventoNotificador();
notificador.agregar(new EmailObservador());
notificador.notificar("Nuevo pedido");`} lang="java" />

### 70. ¿Qué es el patrón Decorator?

**¿Qué es?** Patrón que añade funcionalidad a objetos dinámicamente sin modificar su clase.

**¿Para qué sirve?** Streams de Java, wrappers, extensiones de funcionalidad.

<Code code={`// Component
public interface Cafe {
    double getCosto();
    String getDescripcion();

// Concrete Component
public class CafeSimple implements Cafe {
    public double getCosto() { return 1.0; }
    public String getDescripcion() { return "Café"; }
}

// Decorator
public abstract class CafeDecorator implements Cafe {
    protected Cafe cafe;
    
    public CafeDecorator(Cafe cafe) {
        this.cafe = cafe;
    }
}

// Concrete Decorators
public class ConLeche extends CafeDecorator {
    public ConLeche(Cafe cafe) { super(cafe); }
    
    public double getCosto() { return cafe.getCosto() + 0.5; }
    public String getDescripcion() { return cafe.getDescripcion() + " + Leche"; }
}

public class ConAzucar extends CafeDecorator {
    public ConAzucar(Cafe cafe) { super(cafe); }
    
    public double getCosto() { return cafe.getCosto() + 0.2; }
    public String getDescripcion() { return cafe.getDescripcion() + " + Azúcar"; }
}

// Uso

### 72. ¿Qué es el patrón Proxy?

**¿Qué es?** Patrón que proporciona un sustituto o placeholder para controlar acceso a otro objeto.

**Tipos:** Virtual (lazy loading), Protection (control de acceso), Remote (objetos remotos).

<Code code={`// Subject
public interface Imagen {
    void mostrar();
}

// Real Subject
public class ImagenReal implements Imagen {
    private String archivo;
    
    public ImagenReal(String archivo) {
        this.archivo = archivo;
        cargarDesdeDisc();
    }
    
    private void cargarDesdeDisc() {
        System.out.println("Cargando " + archivo);
    }
    
    @Override
    public void mostrar() {
        System.out.println("Mostrando " + archivo);
    }
}

// Proxy
public class ImagenProxy implements Imagen {
    private ImagenReal imagenReal;
    private String archivo;
    
    public ImagenProxy(String archivo) {
        this.archivo = archivo;
    }
    
    @Override
    public void mostrar() {
        if (imagenReal == null) {
            imagenReal = new ImagenReal(archivo); // Lazy loading
        }
        imagenReal.mostrar();
    }
}`} lang="java" />

### 73. ¿Qué es el patrón Template Method?

**¿Qué es?** Patrón que define el esqueleto de un algoritmo, delegando pasos a subclases.

**¿Para qué sirve?** Reutilizar estructura común, permitir personalización de pasos específicos.

        }

    private void servir() {

        return true;
    }

public class Cafe extends Bebida {

    @Override

### 75. ¿Qué es el patrón State?

**¿Qué es?** Patrón que permite a un objeto cambiar su comportamiento cuando cambia su estado interno.

**¿Para qué sirve?** Máquinas de estado, workflows, evitar condicionales complejos.

<Code code={`// State interface
public interface Estado {
    void insertar();
    void expulsar();
    void dispensar();
}

// Context
public class MaquinaExpendedora {
    private Estado estado;
    
    public void setEstado(Estado estado) {
        this.estado = estado;
    }
    
    public void insertar() { estado.insertar(); }
    public void expulsar() { estado.expulsar(); }
    public void dispensar() { estado.dispensar(); }
}

// Concrete States
public class SinMoneda implements Estado {
    private MaquinaExpendedora maquina;
    
    @Override
    public void insertar() {
        System.out.println("Moneda insertada");
        maquina.setEstado(new ConMoneda(maquina));
    }
    
    @Override
    public void expulsar() {
        System.out.println("No hay moneda");
    }
    
    @Override
    public void dispensar() {
        System.out.println("Inserte moneda primero");
    }
}`} lang="java" />

### 76. ¿Qué es Dependency Injection?

**¿Qué es?** Patrón donde las dependencias se proporcionan externamente en vez de crearlas internamente.

**Beneficios:** Desacoplamiento, testabilidad, flexibilidad.

<Code code={`// Sin DI (acoplado)
public class ServicioUsuario {
    private RepositorioUsuario repo = new RepositorioUsuarioImpl();
}

// Con DI (desacoplado)

    }
}

// Setter injection
public class ServicioUsuario {
    private RepositorioUsuario repo;

    }

// Uso

### 77. ¿Qué es el principio SOLID?

**¿Qué es?** Cinco principios de diseño orientado a objetos para código mantenible y escalable.

| Principio | Descripción |
|-----------|-------------|
| **S**ingle Responsibility | Una clase, una responsabilidad |
| **O**pen/Closed | Abierto para extensión, cerrado para modificación |
| **L**iskov Substitution | Subtipos deben ser sustituibles por sus tipos base |
| **I**nterface Segregation | Interfaces específicas mejor que una general |
| **D**ependency Inversion | Depender de abstracciones, no de implementaciones |

### 78. ¿Qué es el patrón Repository?

**¿Qué es?** Patrón que abstrae el acceso a datos, proporcionando una colección de objetos de dominio.

**Beneficios:** Desacopla lógica de negocio de persistencia, facilita testing.

<Code code={`// Repository interface
public interface UsuarioRepository {
    Optional<Usuario> findById(Long id);
    List<Usuario> findAll();
    Usuario save(Usuario usuario);
    void delete(Long id);
    List<Usuario> findByNombre(String nombre);
}

// Implementation
public class UsuarioRepositoryImpl implements UsuarioRepository {
    private EntityManager em;
    
    @Override
    public Optional<Usuario> findById(Long id) {
        return Optional.ofNullable(em.find(Usuario.class, id));
    }
    
    @Override
    public List<Usuario> findAll() {
        return em.createQuery("SELECT u FROM Usuario u", Usuario.class)
                 .getResultList();
    }
    
    @Override
    public Usuario save(Usuario usuario) {
        if (usuario.getId() == null) {
            em.persist(usuario);
        } else {
            usuario = em.merge(usuario);
        }
        return usuario;
    }
}`} lang="java" />

### 79. ¿Qué es el patrón Service Layer?

**¿Qué es?** Capa que contiene la lógica de negocio, orquestando operaciones entre repositorios.

**Responsabilidades:** Validaciones, transacciones, coordinación de operaciones.

<Code code={`// Service layer - lógica de negocio
@Service
public class UsuarioService {
    private final UsuarioRepository repository;
    private final EmailService emailService;
    
    public UsuarioService(UsuarioRepository repository, EmailService emailService) {
        this.repository = repository;
        this.emailService = emailService;
    }
    
    @Transactional
    public Usuario registrar(UsuarioDTO dto) {
        // Validaciones de negocio
        if (repository.existsByEmail(dto.getEmail())) {
            throw new EmailDuplicadoException();
        }
        
        // Crear usuario
        Usuario usuario = new Usuario();
        usuario.setNombre(dto.getNombre());
        usuario.setEmail(dto.getEmail());
        usuario.setPassword(encriptar(dto.getPassword()));
        
        // Guardar
        usuario = repository.save(usuario);
        
        // Enviar email de bienvenida
        emailService.enviarBienvenida(usuario);
        
        return usuario;
    }
}`} lang="java" />

### 80. ¿Qué es el patrón DTO?

**¿Qué es?** Objeto para transferir datos entre capas/sistemas sin exponer entidades de dominio.

**¿Para qué sirve?** Seguridad (ocultar campos), performance (solo datos necesarios).

<Code code={`// DTO - Data Transfer Object

    private String email;

## Preguntas de Arquitectura (91-100)

### 91. ¿Qué es una arquitectura de microservicios?

**¿Qué es?** Aplicación dividida en servicios pequeños, independientes, desplegables por separado.

**Comunicación:** REST, gRPC, mensajería (Kafka, RabbitMQ).

### 92. ¿Qué es un API Gateway?

**¿Qué es?** Punto de entrada único para microservicios.

**Funciones:** Routing, autenticación, rate limiting, load balancing, transformación.

### 93. ¿Qué es Circuit Breaker?

**¿Qué es?** Patrón que previene cascada de fallos cortando llamadas a servicios fallidos.

**Estados:** Closed (normal), Open (bloqueado), Half-Open (probando).

### 94. ¿Qué es Event Sourcing?

**¿Qué es?** Almacenar cambios como secuencia de eventos en vez de estado actual.

**Beneficios:** Auditoría completa, reconstruir estado, debugging temporal.

### 95. ¿Qué es CQRS?

**¿Qué es?** Command Query Responsibility Segregation: modelos separados para lectura y escritura.

**¿Cuándo usarlo?** Sistemas con patrones de lectura/escritura muy diferentes.

### 96. ¿Qué es un Message Broker?

**¿Qué es?** Sistema de mensajería asíncrona que desacopla productores y consumidores.

**Ejemplos:** Kafka (streaming), RabbitMQ (colas tradicionales).

### 97. ¿Qué es una base de datos NoSQL?

**¿Qué es?** Bases de datos no relacionales, optimizadas para casos de uso específicos.

**Tipos:** Document (MongoDB), Key-Value (Redis), Column (Cassandra), Graph (Neo4j).

### 98. ¿Qué es caching?

**¿Qué es?** Almacenar datos frecuentes en memoria rápida para reducir latencia.

**Herramientas:** Redis (distribuido), Caffeine (local), EhCache.

### 99. ¿Qué es un load balancer?

**¿Qué es?** Distribuye tráfico entre múltiples instancias de un servicio.

**Algoritmos:** Round Robin, Least Connections, IP Hash, Weighted.

### 100. ¿Qué es observabilidad?

**¿Qué es?** Capacidad de entender el estado interno del sistema.

**Pilares:** Logs, Métricas, Traces. **Herramientas:** OpenTelemetry, Prometheus, Grafana, Jaeger.
