## Fundamentos (1-15)

### 1. ¿Qué es JavaScript?

**¿Qué es?**
Lenguaje de programación interpretado, dinámico y débilmente tipado que se ejecuta en el navegador y en el servidor (Node.js).

**¿Para qué sirve?**
- Crear interactividad en páginas web (validaciones, animaciones, eventos)
- Desarrollar aplicaciones web completas (frontend y backend)
- Construir aplicaciones móviles y de escritorio

**Características clave:**
- **Single-threaded** con event loop para manejar asincronía
- **Multiparadigma**: soporta programación funcional, orientada a objetos y basada en eventos
- **Interpretado**: no requiere compilación previa

### 2. ¿Cuáles son los tipos de datos en JavaScript?

**¿Qué es?**
JavaScript tiene 7 tipos primitivos y tipos no primitivos (objetos).

**Tipos primitivos:**
- **string**: texto (`"Hola"`)
- **number**: números enteros y decimales (`42`, `3.14`)
- **boolean**: verdadero/falso (`true`, `false`)
- **undefined**: variable declarada sin valor
- **null**: ausencia intencional de valor
- **symbol**: identificador único (ES6)
- **bigint**: números enteros grandes (ES2020)

**Tipos no primitivos:**
- **object**: colección de propiedades
- **array**: lista ordenada (es un objeto)
- **function**: bloque de código reutilizable (es un objeto)

**Ejemplo práctico:**

<Code code={`// Primitivos
let string = "Hola";
let number = 42;
let boolean = true;
let undefined_var;
let null_var = null;
let symbol = Symbol('id');
let bigint = 9007199254740991n;

// No primitivos (objetos)
let object = { nombre: "Juan" };
let array = [1, 2, 3];
let function_var = function() {};`} lang="javascript" title="Tipos de datos" />

### 3. ¿Qué es el DOM?

**¿Qué es?**
Document Object Model - API que representa el HTML como un árbol de nodos que JavaScript puede manipular.

**¿Para qué sirve?**
- Leer contenido de elementos HTML
- Modificar texto, atributos y estilos
- Agregar o eliminar elementos dinámicamente
- Responder a eventos del usuario

**Ejemplo práctico:**
```javascript
// Seleccionar elemento
const titulo = document.getElementById('titulo');

// Modificar contenido
titulo.textContent = 'Nuevo título';

// Cambiar estilos
titulo.style.color = 'blue';

// Agregar elemento
const parrafo = document.createElement('p');
document.body.appendChild(parrafo);
```

### 4. ¿Qué es el BOM?

**¿Qué es?**
Browser Object Model - API que permite interactuar con el navegador fuera del documento HTML.

**¿Para qué sirve?**
- **window**: objeto global, contiene todo
- **navigator**: información del navegador y dispositivo
- **location**: URL actual, redirecciones
- **history**: historial de navegación
- **screen**: información de la pantalla

**Ejemplo práctico:**
```javascript
// Redireccionar
window.location.href = 'https://ejemplo.com';

// Detectar navegador
console.log(navigator.userAgent);

// Ir atrás en historial
history.back();
```

### 5. ¿Cómo se declara una variable?

**¿Qué es?**
Formas de crear variables en JavaScript: `var`, `let` y `const`.

**¿Cuál usar?**
- **const** (por defecto): valor que no cambiará
- **let**: cuando necesitas reasignar el valor
- **var**: evitar (tiene problemas de scope)

**Diferencias clave:**

| | var | let | const |
|---|-----|-----|-------|
| Scope | Función | Bloque | Bloque |
| Hoisting | Sí (undefined) | Sí (TDZ) | Sí (TDZ) |
| Reasignable | Sí | Sí | No |

**Ejemplo práctico:**

<Code code={`// const - valor fijo (preferido)
const PI = 3.14159;
const usuario = { nombre: 'Juan' }; // El objeto puede mutar
usuario.nombre = 'Pedro'; // OK

// let - cuando necesitas reasignar
let contador = 0;
contador = contador + 1; // OK

// var - evitar por problemas de scope
var antigua = 'ES5';`} lang="javascript" />

### 6. ¿Qué es el scope?

**¿Qué es?**
Ámbito o contexto donde una variable es accesible. Determina dónde puedes usar una variable.

**Tipos de scope:**
- **Global**: accesible en todo el código
- **Función**: solo dentro de la función donde se declara
- **Bloque**: solo dentro de `{}` (let/const)

**Ejemplo práctico:**
```javascript
const global = 'Accesible en todo el código';

function ejemplo() {
  const funcionScope = 'Solo dentro de esta función';
  
  if (true) {
    const bloqueScope = 'Solo dentro de este if';
    var varNoRespeta = 'Escapa del bloque'; // Problema de var
  }
  
  console.log(varNoRespeta); // Funciona (problema)
  // console.log(bloqueScope); // Error: no definida
}
```

### 7. ¿Qué es una función de primera clase?

**¿Qué es?**
En JavaScript, las funciones son "ciudadanos de primera clase", lo que significa que se tratan como cualquier otro valor.

**¿Para qué sirve?**
Permite patrones poderosos como callbacks, higher-order functions y programación funcional.

**Puedes:**
- Asignar funciones a variables
- Pasar funciones como argumentos
- Retornar funciones desde otras funciones
- Almacenar funciones en arrays u objetos

**Ejemplo práctico:**
```javascript
// Asignar a variable
const saludar = function(nombre) {
  return `Hola, ${nombre}`;
};

// Pasar como argumento (callback)
const usuarios = ['Ana', 'Juan'];
usuarios.forEach(saludar);

// Retornar función
function multiplicador(factor) {
  return function(numero) {
    return numero * factor;
  };
}
const duplicar = multiplicador(2);
duplicar(5); // 10
```

### 8. ¿Qué es una función pura?

**¿Qué es?**
Función que cumple dos reglas: mismo input siempre produce mismo output, y no tiene efectos secundarios.

**¿Para qué sirve?**
- Código predecible y fácil de testear
- Facilita debugging (sin sorpresas)
- Permite optimizaciones (memoización)

**Ejemplo práctico:**

<Code code={`// PURA: mismo input = mismo output, sin efectos secundarios
function sumar(a, b) {
  return a + b;
}

// Función impura - modifica estado externo
let total = 0;
function sumarImpura(valor) {
  total += valor; // Efecto secundario
  return total;
}`} lang="javascript" />

### 9. ¿Qué es una IIFE?

**¿Qué es?**
Immediately Invoked Function Expression - función que se define y ejecuta inmediatamente.

**¿Para qué sirve?**
- Crear scope privado (variables no contaminan el global)
- Ejecutar código una sola vez al cargar
- Patrón común antes de ES6 modules

**Ejemplo práctico:**

<Code code={`// IIFE clásica
(function() {
  console.log("Se ejecuta inmediatamente");
})();

// Con arrow function
(() => {
  console.log("También funciona");
})();`} lang="javascript" />

### 10. ¿Qué es el strict mode?

**¿Qué es?**
Modo estricto de JavaScript que hace el código más seguro y previene errores silenciosos.

**¿Para qué sirve?**
- Convierte errores silenciosos en excepciones
- Prohíbe sintaxis problemática
- Prepara el código para futuras versiones de JS

**¿Cómo activarlo?**
Agregar `"use strict";` al inicio del archivo o función.

**Ejemplo práctico:**

<Code code={`"use strict";

// Previene errores silenciosos
x = 10; // ReferenceError: x is not defined

// Previene uso de palabras reservadas
let private = 1; // SyntaxError`} lang="javascript" />

### 11. ¿Qué es NaN?

**¿Qué es?**
Not a Number - valor especial que representa un resultado matemático inválido.

**Características importantes:**
- Es de tipo `number` (irónicamente)
- `NaN !== NaN` (es el único valor que no es igual a sí mismo)
- Usar `Number.isNaN()` para verificar (más preciso que `isNaN()`)

**Ejemplo práctico:**

<Code code={`// Operaciones que producen NaN
console.log(0 / 0);           // NaN
console.log(parseInt('abc')); // NaN

// Verificar NaN
console.log(typeof NaN);        // "number"
console.log(NaN === NaN);       // false
console.log(isNaN(NaN));        // true
console.log(Number.isNaN(NaN)); // true (más preciso)`} lang="javascript" />

### 12. ¿Cómo verificar si una variable es un array?

**¿Qué es?**
Métodos para determinar si un valor es un array.

**¿Cuál usar?**
- **`Array.isArray()`**: Método recomendado, funciona siempre
- **`instanceof Array`**: Puede fallar entre iframes
- **`typeof`**: NO funciona (retorna "object")

**Ejemplo práctico:**

<Code code={`const arr = [1, 2, 3];
const obj = { length: 3 }; // Parece array pero no lo es

Array.isArray(arr);           // true (recomendado)
arr instanceof Array;         // true
arr.constructor === Array;    // true`} lang="javascript" />

### 13. ¿Qué es type coercion?

**¿Qué es?**
Conversión automática de un tipo de dato a otro.

**Tipos de coerción:**
- **Implícita**: JavaScript convierte automáticamente (`"5" + 3 = "53"`)
- **Explícita**: Tú conviertes manualmente (`Number("5") = 5`)

**¿Por qué importa?**
Puede causar bugs sutiles si no entiendes las reglas de conversión.

**Ejemplo práctico:**

<Code code={`// Coerción implícita (cuidado con estas)
console.log("5" + 3);   // "53" (string)
console.log("5" - 3);   // 2 (number)
console.log(true + 1);  // 2

// Coerción explícita
Number("5");    // 5
String(5);      // "5"
Boolean(0);     // false`} lang="javascript" />

### 14. ¿Qué son los template literals?

**¿Qué es?**
Strings delimitados por backticks (\`) que permiten interpolación y múltiples líneas.

**¿Para qué sirve?**
- Insertar variables/expresiones con `${...}`
- Crear strings multilínea sin `\n`
- Más legible que concatenación con `+`

**Ejemplo práctico:**

<Code code={`const nombre = "Juan";
const edad = 25;

// Template literal
const mensaje = \`Hola, soy \${nombre} y tengo \${edad} años\`;

// Multilínea
const html = \`
  <div>
    <h1>\${nombre}</h1>
  </div>
\`;`} lang="javascript" />

### 15. ¿Qué es destructuring?

**¿Qué es?**
Sintaxis para extraer valores de arrays u objetos en variables individuales.

**¿Para qué sirve?**
- Código más limpio y legible
- Extraer solo lo que necesitas
- Valores por defecto
- Renombrar variables

**Ejemplo práctico:**

<Code code={`// Array destructuring
const [a, b, ...resto] = [1, 2, 3, 4, 5];
// a = 1, b = 2, resto = [3, 4, 5]

// Object destructuring
const { nombre, edad, ciudad = "Lima" } = { nombre: "Juan", edad: 25 };
// nombre = "Juan", edad = 25, ciudad = "Lima"`} lang="javascript" />

## Funciones y Objetos (16-30)

### 16. ¿Qué es `this`?

**¿Qué es?**
Palabra clave que referencia al contexto de ejecución actual.

**¿Cómo funciona?**
- **En objeto**: referencia al objeto que contiene el método
- **En función normal**: depende de cómo se llama (dinámico)
- **En arrow function**: hereda del scope padre (léxico)
- **En evento DOM**: referencia al elemento que disparó el evento

**Ejemplo práctico:**

<Code code={`// En objeto - referencia al objeto
const obj = {
  nombre: "Juan",
  saludar() {
    console.log(this.nombre);
  }
};

// En función normal - depende de cómo se llama
function mostrar() {
  console.log(this);
}

// En arrow function - hereda del contexto
const arrow = () => console.log(this);`} lang="javascript" />

### 17. ¿Qué son call, apply y bind?

**¿Qué es?**
Métodos para controlar explícitamente el valor de `this` en una función.

**Diferencias:**
- **call**: Ejecuta la función con `this` y argumentos separados
- **apply**: Ejecuta la función con `this` y argumentos en array
- **bind**: Retorna una NUEVA función con `this` fijo (no ejecuta)

**¿Cuándo usar cada uno?**
- `call`/`apply`: Ejecutar inmediatamente con otro contexto
- `bind`: Crear función para usar después (ej: event handlers)

**Ejemplo práctico:**

<Code code={`function saludar(saludo, signo) {
  console.log(\`\${saludo}, \${this.nombre}\${signo}\`);
}

const persona = { nombre: "Juan" };

// call - argumentos separados
saludar.call(persona, "Hola", "!");

// apply - argumentos en array
saludar.apply(persona, ["Hola", "!"]);

// bind - retorna nueva función
const saludarJuan = saludar.bind(persona);
saludarJuan("Hola", "!");`} lang="javascript" />

### 18. ¿Qué es el prototype?

**¿Qué es?**
Objeto del que otros objetos heredan propiedades y métodos. Es el mecanismo de herencia en JavaScript.

**Conceptos clave:**
- **`prototype`**: Propiedad de funciones constructoras
- **`__proto__`**: Referencia al prototipo del objeto (interno)
- **Cadena de prototipos**: Búsqueda de propiedades hacia arriba

**¿Para qué sirve?**
Compartir métodos entre todas las instancias (ahorra memoria).

**Ejemplo práctico:**

<Code code={`function Persona(nombre) {
  this.nombre = nombre;
}

Persona.prototype.saludar = function() {
  console.log(\`Hola, soy \${this.nombre}\`);
};

const juan = new Persona("Juan");
juan.saludar(); // "Hola, soy Juan"`} lang="javascript" />

### 19. ¿Qué es la herencia prototípica?

**¿Qué es?**
Mecanismo donde objetos heredan directamente de otros objetos a través de la cadena de prototipos.

**¿Cómo funciona?**
1. Cuando accedes a una propiedad, JS la busca en el objeto
2. Si no la encuentra, busca en su prototipo
3. Continúa subiendo hasta llegar a `null`

**Clases ES6:**
Son "azúcar sintáctico" sobre prototipos (mismo mecanismo, sintaxis más clara).

**Ejemplo práctico:**

<Code code={`// ES6 Classes (azúcar sintáctico sobre prototipos)
class Animal {
  constructor(nombre) {
    this.nombre = nombre;
  }
  hablar() {
    console.log(\`\${this.nombre} hace un sonido\`);
  }
}

class Perro extends Animal {
  hablar() {
    console.log(\`\${this.nombre} ladra\`);
  }
}`} lang="javascript" />

### 20. ¿Qué es Object.freeze vs Object.seal?

**¿Qué es?**
Métodos para controlar la mutabilidad de objetos.

**Diferencias:**

| Método | Agregar | Modificar | Eliminar |
|--------|---------|-----------|----------|
| `freeze` | ❌ | ❌ | ❌ |
| `seal` | ❌ | ✅ | ❌ |
| Normal | ✅ | ✅ | ✅ |

**Nota:** Ambos son "shallow" (no afectan objetos anidados).

**Ejemplo práctico:**

<Code code={`const obj = { a: 1, b: 2 };

// freeze - no se puede modificar nada
Object.freeze(obj);
obj.a = 10;     // No funciona
obj.c = 3;      // No funciona
delete obj.a;   // No funciona

// seal - se pueden modificar valores existentes
const obj2 = { a: 1, b: 2 };
Object.seal(obj2);
obj2.a = 10;    // Funciona
obj2.c = 3;     // No funciona
delete obj2.a;  // No funciona`} lang="javascript" />

### 21. ¿Cómo clonar un objeto?

**¿Qué es?**
Crear una copia independiente de un objeto.

**Tipos de copia:**
- **Shallow (superficial)**: Copia el primer nivel, objetos anidados siguen siendo referencias
- **Deep (profunda)**: Copia todo, incluyendo objetos anidados

**Métodos:**
- Shallow: `{...obj}`, `Object.assign()`
- Deep: `structuredClone()` (moderno), `JSON.parse(JSON.stringify())`

**Ejemplo práctico:**

<Code code={`const original = { a: 1, b: { c: 2 } };

// Shallow copy
const copia1 = { ...original };
const copia2 = Object.assign({}, original);

// Deep copy
const copiaDeep = JSON.parse(JSON.stringify(original));
const copiaDeep2 = structuredClone(original); // Moderno`} lang="javascript" />

### 22. ¿Qué es el optional chaining?

**¿Qué es?**
Operador `?.` que permite acceder a propiedades anidadas de forma segura.

**¿Para qué sirve?**
- Evita errores "Cannot read property of undefined"
- Reemplaza múltiples chequeos de null/undefined
- Retorna `undefined` si algo en la cadena no existe

**Variantes:**
- `obj?.prop` - Acceso a propiedad
- `obj?.[expr]` - Acceso dinámico
- `func?.()` - Llamada a función

**Ejemplo práctico:**

<Code code={`const usuario = {
  nombre: "Juan",
  direccion: {
    ciudad: "Lima"
  }
};

// Sin optional chaining
const pais = usuario.direccion && usuario.direccion.pais;

// Con optional chaining
const pais2 = usuario.direccion?.pais; // undefined
const metodo = usuario.saludar?.(); // undefined`} lang="javascript" />

### 23. ¿Qué es el nullish coalescing?

**¿Qué es?**
Operador `??` que proporciona un valor por defecto solo cuando el valor es `null` o `undefined`.

**Diferencia con `||`:**
- `||` considera falsy: `0`, `""`, `false`, `null`, `undefined`
- `??` solo considera: `null`, `undefined`

**¿Cuándo usar `??`?**
Cuando `0` o `""` son valores válidos que no quieres reemplazar.

**Ejemplo práctico:**

<Code code={`// || considera falsy: 0, "", false, null, undefined
const a = 0 || "default";  // "default"

// ?? solo considera null y undefined
const b = 0 ?? "default";  // 0
const c = null ?? "default"; // "default"`} lang="javascript" />

### 24. ¿Qué son los getters y setters?

**¿Qué es?**
Métodos especiales que se ejecutan automáticamente al leer o escribir una propiedad.

**¿Para qué sirve?**
- **get**: Ejecuta lógica al LEER (ej: formatear, calcular)
- **set**: Ejecuta lógica al ESCRIBIR (ej: validar, transformar)

**Casos de uso:**
- Validación de datos
- Propiedades calculadas
- Encapsulación

**Ejemplo práctico:**

<Code code={`const persona = {
  _nombre: "",
  
  get nombre() {
    return this._nombre.toUpperCase();
  },
  
  set nombre(valor) {
    this._nombre = valor.trim();
  }
};

persona.nombre = "  juan  ";
console.log(persona.nombre); // "JUAN"`} lang="javascript" />

### 25. ¿Qué es un Symbol?

**¿Qué es?**
Tipo primitivo que crea identificadores únicos e inmutables.

**¿Para qué sirve?**
- Crear claves de propiedades que no colisionen
- Propiedades "ocultas" (no aparecen en `for...in` ni `Object.keys()`)
- Definir comportamientos especiales (`Symbol.iterator`)

**Características:**
- Cada Symbol es único, incluso con la misma descripción
- No se convierte automáticamente a string

**Ejemplo práctico:**

<Code code={`const id = Symbol('id');
const id2 = Symbol('id');

console.log(id === id2); // false (siempre únicos)

const obj = {
  [id]: 123,
  nombre: "Juan"
};

console.log(obj[id]); // 123
console.log(Object.keys(obj)); // ["nombre"] - Symbol no aparece`} lang="javascript" />

### 26. ¿Qué es un Map vs Object?

**¿Qué es?**
Map es una colección de pares clave-valor, similar a Object pero con diferencias importantes.

**Diferencias:**

| Característica | Map | Object |
|---------------|-----|--------|
| Tipo de clave | Cualquiera | Solo string/symbol |
| Orden | Mantiene orden de inserción | No garantizado |
| Tamaño | `map.size` | `Object.keys().length` |
| Iteración | Directamente iterable | Necesita `Object.keys()` |

**¿Cuándo usar Map?**
Cuando necesitas claves que no sean strings o el orden importa.

**Ejemplo práctico:**

<Code code={`// Map - cualquier tipo como clave
const map = new Map();
map.set({ id: 1 }, "valor");
map.set(1, "número");

// Object - solo strings/symbols como clave
const obj = {};
obj[{ id: 1 }] = "valor"; // Se convierte a "[object Object]"`} lang="javascript" />

### 27. ¿Qué es un Set?

**¿Qué es?**
Colección de valores únicos (no permite duplicados).

**¿Para qué sirve?**
- Eliminar duplicados de un array
- Verificar pertenencia rápidamente (`has()` es O(1))
- Operaciones de conjuntos (unión, intersección)

**Métodos principales:**
- `add(valor)`: Agregar
- `has(valor)`: Verificar existencia
- `delete(valor)`: Eliminar
- `size`: Cantidad de elementos

**Ejemplo práctico:**

<Code code={`// Eliminar duplicados
const set = new Set([1, 2, 2, 3, 3, 3]);
console.log([...set]); // [1, 2, 3]

set.add(4);
set.has(2);    // true
set.delete(1);
set.size;      // 3`} lang="javascript" />

### 28. ¿Qué es un WeakMap y WeakSet?

**¿Qué es?**
Versiones "débiles" de Map y Set que permiten garbage collection de sus claves/valores.

**Características:**
- Las claves DEBEN ser objetos (no primitivos)
- No son iterables (no puedes recorrerlos)
- No tienen propiedad `size`
- Cuando la clave ya no tiene referencias, se elimina automáticamente

**¿Para qué sirve?**
- Almacenar metadata asociada a objetos sin prevenir su eliminación
- Cachés que se limpian automáticamente

**Ejemplo práctico:**

<Code code={`// WeakMap - claves son objetos, permite garbage collection
const wm = new WeakMap();
let obj = { data: "importante" };
wm.set(obj, "metadata");

obj = null; // El objeto puede ser recolectado

// WeakSet - similar pero para valores únicos
const ws = new WeakSet();`} lang="javascript" />

### 29. ¿Qué es un Proxy?

**¿Qué es?**
Objeto que envuelve a otro y permite interceptar operaciones sobre él.

**¿Para qué sirve?**
- Validación de datos al asignar propiedades
- Logging de accesos
- Valores por defecto automáticos
- Crear objetos "reactivos" (como en Vue.js)

**Operaciones interceptables (traps):**
`get`, `set`, `has`, `deleteProperty`, `apply`, etc.

**Ejemplo práctico:**

<Code code={`const handler = {
  get(target, prop) {
    console.log(\`Accediendo a \${prop}\`);
    return target[prop];
  },
  set(target, prop, value) {
    console.log(\`Estableciendo \${prop} = \${value}\`);
    target[prop] = value;
    return true;
  }
};

const proxy = new Proxy({}, handler);
proxy.nombre = "Juan"; // "Estableciendo nombre = Juan"
console.log(proxy.nombre); // "Accediendo a nombre" → "Juan"`} lang="javascript" />

### 30. ¿Qué es Reflect?

**¿Qué es?**
API con métodos estáticos para operaciones de objetos, diseñada para complementar Proxy.

**¿Para qué sirve?**
- Realizar operaciones de objetos de forma funcional
- Dentro de Proxy handlers, delegar al comportamiento original
- Retorna booleanos en vez de lanzar errores

**Métodos principales:**
`Reflect.get()`, `Reflect.set()`, `Reflect.has()`, `Reflect.deleteProperty()`

**Ejemplo práctico:**

<Code code={`const obj = { a: 1 };

// Métodos de Reflect
Reflect.get(obj, 'a');           // 1
Reflect.set(obj, 'b', 2);        // true
Reflect.has(obj, 'a');           // true
Reflect.deleteProperty(obj, 'a'); // true`} lang="javascript" />

## Asincronía (31-40)

### 31. ¿Qué es un callback?

**¿Qué es?**
Función que se pasa como argumento a otra función para ejecutarse después.

**¿Para qué sirve?**
- Manejar operaciones asíncronas (antes de Promises)
- Personalizar comportamiento de funciones
- Event handlers

**Problema:**
Anidación excesiva lleva a "callback hell".

**Ejemplo práctico:**

<Code code={`function obtenerDatos(callback) {
  setTimeout(() => {
    callback({ nombre: "Juan" });
  }, 1000);
}

obtenerDatos((data) => {
  console.log(data);
});`} lang="javascript" />

### 32. ¿Qué es callback hell?

**¿Qué es?**
Anidación excesiva de callbacks que hace el código difícil de leer y mantener.

**Problemas:**
- Código en forma de "pirámide"
- Difícil de debuggear
- Manejo de errores complicado

**Soluciones:**
- Promises (`.then()` encadenado)
- async/await (sintaxis síncrona)

**Ejemplo del problema:**

<Code code={`// Callback hell - difícil de leer y mantener
obtenerUsuario(id, (usuario) => {
  obtenerPedidos(usuario.id, (pedidos) => {
    obtenerDetalles(pedidos[0].id, (detalles) => {
      // ... más anidación
    });
  });
});`} lang="javascript" />

### 33. ¿Cuáles son los estados de una Promise?

**¿Qué es?**
Una Promise tiene 3 estados posibles durante su ciclo de vida.

**Estados:**
- **pending**: Estado inicial, esperando resultado
- **fulfilled**: Operación exitosa, tiene un valor
- **rejected**: Operación fallida, tiene un error

**Importante:**
Una vez que cambia de `pending`, NO puede volver a cambiar (inmutable).

**Ejemplo práctico:**

<Code code={`// Estados: pending, fulfilled, rejected
const promise = new Promise((resolve, reject) => {
  // pending...
  if (exito) {
    resolve(valor);  // fulfilled
  } else {
    reject(error);   // rejected
  }
});`} lang="javascript" />

### 34. ¿Qué es Promise.all?

**¿Qué es?**
Método que espera a que TODAS las promises se resuelvan.

**Comportamiento:**
- Retorna array con todos los resultados (en orden)
- Si UNA falla, TODO falla inmediatamente
- Ejecuta en paralelo (no secuencial)

**¿Cuándo usarlo?**
Cuando necesitas todos los resultados y uno fallido invalida todo.

**Ejemplo práctico:**

<Code code={`const p1 = fetch('/api/users');
const p2 = fetch('/api/posts');
const p3 = fetch('/api/comments');

// Espera a que TODAS se resuelvan
const [users, posts, comments] = await Promise.all([p1, p2, p3]);

// Si una falla, todo falla`} lang="javascript" />

### 35. ¿Qué es Promise.race?

**¿Qué es?**
Método que retorna la PRIMERA promise que se resuelva O rechace.

**¿Para qué sirve?**
- Implementar timeouts
- Obtener el resultado más rápido de múltiples fuentes

**Comportamiento:**
La primera en terminar (exitosa o fallida) determina el resultado.

**Ejemplo práctico - Timeout:**

<Code code={`// Retorna la primera que se resuelva o rechace
const resultado = await Promise.race([
  fetch('/api/rapido'),
  new Promise((_, reject) => 
    setTimeout(() => reject('Timeout'), 5000)
  )
]);`} lang="javascript" />

### 36. ¿Qué es Promise.allSettled?

**¿Qué es?**
Método que espera a que TODAS las promises terminen, sin importar si fallan.

**Diferencia con Promise.all:**
- `all`: Falla si UNA falla
- `allSettled`: Siempre espera a todas

**Retorna:**
Array de objetos con `{ status, value }` o `{ status, reason }`.

**¿Cuándo usarlo?**
Cuando quieres saber el resultado de todas, incluso las fallidas.

**Ejemplo práctico:**

<Code code={`const resultados = await Promise.allSettled([
  Promise.resolve(1),
  Promise.reject('error'),
  Promise.resolve(3)
]);

// [
//   { status: 'fulfilled', value: 1 },
//   { status: 'rejected', reason: 'error' },
//   { status: 'fulfilled', value: 3 }
// ]`} lang="javascript" />

### 37. ¿Qué es Promise.any?

**¿Qué es?**
Método que retorna la PRIMERA promise que se resuelva EXITOSAMENTE.

**Diferencia con Promise.race:**
- `race`: Primera en terminar (exitosa o fallida)
- `any`: Primera en tener ÉXITO (ignora rechazos)

**Comportamiento:**
Solo falla si TODAS las promises fallan (AggregateError).

**Ejemplo práctico:**

<Code code={`// Retorna la primera que se RESUELVA
const primera = await Promise.any([
  Promise.reject('error1'),
  Promise.resolve('éxito'),
  Promise.reject('error2')
]);
// 'éxito'`} lang="javascript" />

### 38. ¿Cómo manejar errores en async/await?

**¿Qué es?**
Formas de capturar errores cuando usas async/await.

**Método principal: try/catch**
Envuelve el código asíncrono y captura tanto errores síncronos como rechazos de await.

**Alternativa: .catch() al final**
```javascript
async function getData() { ... }
getData().catch(error => console.error(error));
```

**Ejemplo práctico:**

<Code code={`// Con try/catch
async function obtenerDatos() {
  try {
    const response = await fetch('/api/data');
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Error:', error);
    throw error;
  }
}`} lang="javascript" />

### 39. ¿Qué son los microtasks y macrotasks?

**¿Qué es?**
Dos colas de tareas en el event loop con diferentes prioridades.

**Microtasks (alta prioridad):**
- Promises (`.then`, `.catch`, `.finally`)
- `queueMicrotask()`
- MutationObserver

**Macrotasks (baja prioridad):**
- `setTimeout`, `setInterval`
- I/O, eventos del DOM
- `requestAnimationFrame`

**Orden de ejecución:**
1. Código síncrono
2. TODOS los microtasks
3. UN macrotask
4. Repetir

**Ejemplo práctico:**

<Code code={`// Macrotasks: setTimeout, setInterval, I/O
// Microtasks: Promises, queueMicrotask, MutationObserver

console.log('1');
setTimeout(() => console.log('2'), 0);  // Macrotask
Promise.resolve().then(() => console.log('3')); // Microtask
console.log('4');

// Salida: 1, 4, 3, 2`} lang="javascript" />

### 40. ¿Qué es un generator?

**¿Qué es?**
Función especial que puede pausar y reanudar su ejecución.

**Sintaxis:**
- Declaración: `function* nombre() { }`
- Pausar: `yield valor`
- Reanudar: `gen.next()`

**¿Para qué sirve?**
- Crear iteradores personalizados
- Generar secuencias infinitas
- Control de flujo asíncrono (con co/redux-saga)

**Ejemplo práctico:**

<Code code={`function* generador() {
  yield 1;
  yield 2;
  yield 3;
}

const gen = generador();
console.log(gen.next()); // { value: 1, done: false }
console.log(gen.next()); // { value: 2, done: false }
console.log(gen.next()); // { value: 3, done: false }
console.log(gen.next()); // { value: undefined, done: true }`} lang="javascript" />

### 41. ¿Cómo seleccionar elementos del DOM?

**¿Qué es?**
Métodos para obtener referencias a elementos HTML desde JavaScript.

**Métodos principales:**
- **`querySelector()`**: Retorna el PRIMER elemento que coincida (CSS selector)
- **`querySelectorAll()`**: Retorna TODOS los elementos (NodeList)
- **`getElementById()`**: Por ID (más rápido)
- **`getElementsByClassName()`**: Por clase (colección viva)

**Recomendación:** Preferir `querySelector`/`querySelectorAll` por su flexibilidad.

**Ejemplo práctico:**

document.getElementById('id');

document.querySelectorAll('div');

### 42. ¿Qué es event bubbling y capturing?

**¿Qué es?**
Dos fases de propagación de eventos en el DOM.

**Fases:**
1. **Capturing**: El evento BAJA desde `window` hasta el elemento target
2. **Target**: El evento llega al elemento clickeado
3. **Bubbling**: El evento SUBE desde el target hasta `window`

**Por defecto:** Los listeners se ejecutan en fase de bubbling.

**Controlar con tercer parámetro:**
- `false` o sin parámetro: Bubbling
- `true`: Capturing

**Ejemplo práctico:**

<Code code={`// Bubbling (por defecto): hijo → padre
// Capturing: padre → hijo

element.addEventListener('click', handler, false); // Bubbling
element.addEventListener('click', handler, true);  // Capturing

// Detener propagación
event.stopPropagation();`} lang="javascript" />

### 43. ¿Qué es event delegation?

**¿Qué es?**
Técnica que pone UN solo listener en el padre en vez de muchos en los hijos.

**¿Para qué sirve?**
- Mejor rendimiento (menos listeners)
- Funciona con elementos dinámicos (agregados después)
- Código más limpio

**¿Cómo funciona?**
Usa bubbling + `e.target` para identificar qué elemento hijo disparó el evento.

**Ejemplo práctico:**

<Code code={`// En lugar de agregar listener a cada <li>
document.querySelector('ul').addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {
    console.log('Click en:', e.target.textContent);
  }
});`} lang="javascript" />

### 44. ¿Cuál es la diferencia entre preventDefault y stopPropagation?

**¿Qué es?**
Dos métodos para controlar el comportamiento de eventos.

**Diferencias:**

| Método | Qué hace |
|--------|----------|
| `preventDefault()` | Evita la acción por defecto del navegador |
| `stopPropagation()` | Evita que el evento se propague a otros elementos |

**Ejemplos de acciones por defecto:**
- Submit de formulario
- Click en link (navegar)
- Click derecho (menú contextual)

**Ejemplo práctico:**

<Code code={`// preventDefault - evita comportamiento por defecto
form.addEventListener('submit', (e) => {
  e.preventDefault(); // No envía el formulario
});

// stopPropagation - evita que el evento suba/baje
button.addEventListener('click', (e) => {
  e.stopPropagation(); // No llega al padre
});`} lang="javascript" />

### 45. ¿Qué es el localStorage y sessionStorage?

**¿Qué es?**
APIs para almacenar datos en el navegador del usuario.

**Diferencias:**

| | localStorage | sessionStorage |
|---|--------------|----------------|
| Persistencia | Hasta borrado manual | Hasta cerrar pestaña |
| Compartido | Entre pestañas | Solo esa pestaña |
| Capacidad | ~5-10 MB | ~5-10 MB |

**Importante:** Solo almacenan strings. Usar `JSON.stringify()`/`JSON.parse()` para objetos.

**Ejemplo práctico:**

<Code code={`// localStorage - persiste hasta que se borre
localStorage.setItem('usuario', JSON.stringify({ nombre: 'Juan' }));
const usuario = JSON.parse(localStorage.getItem('usuario'));
localStorage.removeItem('usuario');
localStorage.clear();

// sessionStorage - se borra al cerrar pestaña
sessionStorage.setItem('temp', 'valor');`} lang="javascript" />

### 46. ¿Qué es debounce?

**¿Qué es?**
Técnica que retrasa la ejecución de una función hasta que pase un tiempo sin nuevas llamadas.

**¿Para qué sirve?**
- Búsqueda mientras escribes (esperar a que termine de escribir)
- Redimensionar ventana (ejecutar al final)
- Autoguardado

**¿Cómo funciona?**
Cada llamada reinicia el temporizador. Solo ejecuta cuando "para" de llamar.

**Ejemplo práctico:**

<Code code={`function debounce(func, delay) {
  let timeoutId;
  return function(...args) {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => func.apply(this, args), delay);
  };
}

// Uso: ejecutar solo después de que el usuario deje de escribir
const buscar = debounce((query) => {
  fetch(\`/api/search?q=\${query}\`);
}, 300);`} lang="javascript" />

### 47. ¿Qué es throttle?

**¿Qué es?**
Técnica que limita la ejecución a máximo una vez cada X milisegundos.

**Diferencia con debounce:**
- **Debounce**: Espera a que "pare" de llamar
- **Throttle**: Ejecuta cada X ms mientras se llama

**¿Para qué sirve?**
- Scroll infinito (cargar más contenido)
- Tracking de posición del mouse
- Eventos de resize

**Ejemplo práctico:**

<Code code={`function throttle(func, limit) {
  let inThrottle;
  return function(...args) {
    if (!inThrottle) {
      func.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

// Uso: ejecutar máximo una vez cada X ms
const handleScroll = throttle(() => {
  console.log('Scroll');
}, 100);`} lang="javascript" />

### 48. ¿Qué es el Fetch API?

**Ejemplo práctico:**

<Code code={`// GET

const data = await response.json();

// POST

### 50. ¿Qué es el patrón módulo revelador?

**¿Qué es?**
Patrón de diseño que usa closures para crear encapsulación (variables privadas).

**¿Para qué sirve?**
- Ocultar implementación interna
- Exponer solo una API pública
- Evitar colisiones de nombres

**¿Cómo funciona?**
Una IIFE retorna un objeto con los métodos públicos, las variables internas quedan en el closure.

**Nota:** Antes de ES6 modules, era la forma principal de crear módulos.

**Ejemplo práctico:**

<Code code={`const miModulo = (function() {
  // Privado
  let contador = 0;
  
  function incrementarPrivado() {
    contador++;
  }
  
  // Público (revelado)
  return {
    incrementar: function() {
      incrementarPrivado();
    },
    obtenerValor: function() {
      return contador;
    }
  };
})();`} lang="javascript" />
