## Fundamentos (1-15)

### 1. ¿Qué es JSX?

> Extensión de sintaxis que combina HTML con JavaScript. Se transpila a React.createElement(). No es HTML, es JavaScript.

<Code code={`// JSX
const element = <h1 className="titulo">Hola</h1>;

// Se transpila a:
const element = React.createElement('h1', { className: 'titulo' }, 'Hola');`} lang="jsx" />

### 2. ¿Por qué usamos className en vez de class?

> `class` es palabra reservada en JS. JSX usa `className` para atributo CSS y `htmlFor` en vez de `for`.

### 3. ¿Qué son los Fragments?

> Permiten agrupar elementos sin agregar nodo extra al DOM. Sintaxis: `<React.Fragment>` o `<>`.

<Code code={`// Problema: React necesita un elemento padre
// Solución: Fragment (no agrega nodo al DOM)

// Sintaxis completa
<React.Fragment>
  <h1>Título</h1>
  <p>Párrafo</p>
</React.Fragment>

// Sintaxis corta
<>
  <h1>Título</h1>
  <p>Párrafo</p>
</>`} lang="jsx" />

### 4. ¿Qué es el renderizado condicional?

> Mostrar componentes según condición. Usar: `&&` (si true), ternario `? :`, o return temprano.

<Code code={`function Componente({ isLoggedIn, items }) {
  // Con &&
  return <div>{isLoggedIn && <UserPanel />}</div>;
  
  // Con ternario
  return <div>{isLoggedIn ? <UserPanel /> : <Login />}</div>;
  
  // Con if temprano
  if (!isLoggedIn) return <Login />;
  return <UserPanel />;
  
  // Con variable
  let content;
  if (isLoggedIn) {
    content = <UserPanel />;
  } else {
    content = <Login />;
  }
  return <div>{content}</div>;
}`} lang="jsx" />

<Code code={`function Lista({ items }) {

        <li key={item.id}>

### 6. ¿Qué es prop drilling y cómo evitarlo?

> Pasar props a través de muchos niveles. Soluciones: Context API, estado global (Redux/Zustand), composición.

<Code code={`// Prop drilling - pasar props a través de muchos niveles
<Abuelo user={user}>
  <Padre user={user}>
    <Hijo user={user}>
      <Nieto user={user} /> {/* Finalmente lo usa */}
    </Hijo>
  </Padre>
</Abuelo>

// Soluciones:
// 1. Context API
// 2. Estado global (Redux, Zustand)
// 3. Composición de componentes`} lang="jsx" />

### 7. ¿Qué es Context API?

> Sistema para compartir datos sin prop drilling. createContext + Provider + useContext. Para datos globales (tema, usuario).

<Code code={`// 1. Crear contexto
const UserContext = createContext(null);

// 2. Proveer valor
function App() {
  const [user, setUser] = useState({ nombre: 'Juan' });
  
  return (
    <UserContext.Provider value={{ user, setUser }}>
      <MiApp />
    </UserContext.Provider>
  );
}

// 3. Consumir en cualquier nivel
function Nieto() {
  const { user } = useContext(UserContext);
  return <p>Hola, {user.nombre}</p>;
}`} lang="jsx" />

### 8. ¿Cuándo usar Context vs Redux?

**¿Qué es?**
Context es la solución nativa de React para compartir estado global. Redux es una librería externa más robusta.

**¿Para qué sirve?**
- **Context**: Ideal para datos que cambian poco (tema, idioma, usuario autenticado)
- **Redux**: Ideal para estado complejo con muchas actualizaciones y lógica de negocio

**¿Cómo elegir?**

| Context | Redux |
|---------|-------|
| Estado simple | Estado complejo |
| Pocos consumidores | Muchos consumidores |
| Actualizaciones poco frecuentes | Actualizaciones frecuentes |
| Sin middleware | Necesitas middleware |

### 9. ¿Qué es useRef?

**¿Qué es?**
Hook que crea una referencia mutable que persiste durante todo el ciclo de vida del componente.

**¿Para qué sirve?**
1. **Acceder a elementos del DOM** directamente (focus, scroll, medidas)
2. **Guardar valores** que no deben causar re-render al cambiar (contadores, timers, valores previos)

**¿Cómo se usa?**
- `useRef(valorInicial)` retorna un objeto `{ current: valorInicial }`
- Cambiar `.current` NO causa re-render
- El valor persiste entre renders

**Ejemplo práctico - Enfocar un input automáticamente:**

<Code code={`function FormularioBusqueda() {
  const inputRef = useRef(null);
  
  // Al montar, enfoca el input
  useEffect(() => {
    inputRef.current.focus();
  }, []);
  
  return (
    <input 
      ref={inputRef} 
      placeholder="Buscar..." 
    />
  );
}`} lang="jsx" />

**Ejemplo práctico - Guardar valor anterior:**

<Code code={`function Contador() {
  const [count, setCount] = useState(0);
  const prevCount = useRef(0);
  
  useEffect(() => {
    prevCount.current = count; // Guarda el valor anterior
  }, [count]);
  
  return (
    <div>
      <p>Actual: {count}</p>
      <p>Anterior: {prevCount.current}</p>
      <button onClick={() => setCount(c => c + 1)}>+1</button>
    </div>
  );
}`} lang="jsx" />

### 10. ¿Qué es useCallback?

**¿Qué es?**
Hook que memoriza una función para que mantenga la misma referencia entre renders.

**¿Para qué sirve?**
Evitar que componentes hijos se re-rendericen innecesariamente cuando les pasas funciones como props.

**¿Cuándo usarlo?**
- Cuando pasas funciones a componentes hijos optimizados con `React.memo`
- Cuando la función es dependencia de un `useEffect`

**¿Cómo se usa?**
```jsx
const funcionMemorizada = useCallback(() => {
  // lógica
}, [dependencias]);
```

**Ejemplo práctico - Botón que no se re-renderiza innecesariamente:**

<Code code={`// Componente hijo optimizado
const Boton = React.memo(({ onClick, texto }) => {
  console.log('Boton renderizado');
  return <button onClick={onClick}>{texto}</button>;
});

function Formulario() {
  const [nombre, setNombre] = useState('');
  const [contador, setContador] = useState(0);
  
  // SIN useCallback: Boton se re-renderiza cada vez que escribes
  // const handleSubmit = () => console.log('Enviado:', nombre);
  
  // CON useCallback: Boton NO se re-renderiza al escribir
  const handleSubmit = useCallback(() => {
    console.log('Enviado:', nombre);
  }, [nombre]); // Solo cambia si 'nombre' cambia
  
  return (
    <div>
      <input 
        value={nombre} 
        onChange={(e) => setNombre(e.target.value)} 
      />
      <Boton onClick={handleSubmit} texto="Enviar" />
      <p>Contador: {contador}</p>
      <button onClick={() => setContador(c => c + 1)}>+1</button>
    </div>
  );
}`} lang="jsx" />

### 11. ¿Qué es useMemo?

**¿Qué es?**
Hook que memoriza el **resultado** de un cálculo para evitar recalcularlo en cada render.

**¿Para qué sirve?**
Optimizar rendimiento cuando tienes cálculos costosos que no necesitan ejecutarse en cada render.

**¿Cuándo usarlo?**
- Cálculos matemáticos complejos
- Filtrar/ordenar listas grandes
- Transformar datos
- Crear objetos que serán props de componentes memorizados

**¿Cómo se usa?**
```jsx
const valorMemorizado = useMemo(() => {
  return calculoCostoso(a, b);
}, [a, b]); // Solo recalcula si a o b cambian
```

**Ejemplo práctico - Calcular precio total con descuento:**

<Code code={`function CarritoCompras({ productos, descuento }) {
  // Sin useMemo: se recalcula en CADA render
  // const total = productos.reduce((sum, p) => sum + p.precio * p.cantidad, 0);
  
  // Con useMemo: solo recalcula si productos o descuento cambian
  const totalConDescuento = useMemo(() => {
    console.log('Calculando total...'); // Verás esto solo cuando cambie
    
    const subtotal = productos.reduce(
      (sum, producto) => sum + producto.precio * producto.cantidad, 
      0
    );
    
    const descuentoAplicado = subtotal * (descuento / 100);
    return subtotal - descuentoAplicado;
  }, [productos, descuento]);
  
  return (
    <div>
      <h2>Tu Carrito</h2>
      {productos.map(p => (
        <p key={p.id}>{p.nombre}: \${p.precio} x {p.cantidad}</p>
      ))}
      <hr />
      <p><strong>Total: \${totalConDescuento.toFixed(2)}</strong></p>
    </div>
  );
}`} lang="jsx" />

**Ejemplo práctico - Filtrar lista grande:**

<Code code={`function ListaUsuarios({ usuarios, filtro }) {
  // Solo filtra cuando 'usuarios' o 'filtro' cambian
  const usuariosFiltrados = useMemo(() => {
    return usuarios.filter(user => 
      user.nombre.toLowerCase().includes(filtro.toLowerCase())
    );
  }, [usuarios, filtro]);
  
  return (
    <ul>
      {usuariosFiltrados.map(user => (
        <li key={user.id}>{user.nombre}</li>
      ))}
    </ul>
  );
}`} lang="jsx" />

### 12. ¿Cuál es la diferencia entre useMemo y useCallback?

**Diferencia clave:**
- **useMemo**: Memoriza el **RESULTADO** de ejecutar una función
- **useCallback**: Memoriza la **FUNCIÓN** en sí misma

**Analogía simple:**
- `useMemo` = "Recuerda la respuesta" (ej: el número 42)
- `useCallback` = "Recuerda cómo calcular" (ej: la fórmula)

**¿Cuándo usar cada uno?**

| useMemo | useCallback |
|---------|-------------|
| Cálculos costosos | Funciones como props |
| Transformar datos | Event handlers |
| Valores derivados | Dependencias de useEffect |

**Ejemplo comparativo:**

<Code code={`function Ejemplo({ items }) {
  // useMemo - memoriza el RESULTADO (un número)
  const total = useMemo(() => {
    return items.reduce((sum, item) => sum + item.precio, 0);
  }, [items]);
  // total = 150 (el valor calculado)
  
  // useCallback - memoriza la FUNCIÓN
  const calcularTotal = useCallback(() => {
    return items.reduce((sum, item) => sum + item.precio, 0);
  }, [items]);
  // calcularTotal = () => { ... } (la función)
  // Para obtener el valor: calcularTotal() = 150
  
  return (
    <div>
      <p>Total directo: {total}</p>
      <p>Total llamando función: {calcularTotal()}</p>
      <BotonPago onPagar={calcularTotal} />
    </div>
  );
}`} lang="jsx" />

### 13. ¿Qué es React.memo?

**¿Qué es?**
HOC (Higher Order Component) que memoriza un componente para evitar re-renders innecesarios.

**¿Para qué sirve?**
Evitar que un componente se re-renderice si sus props no han cambiado.

**¿Cuándo usarlo?**
- Componentes que renderizan frecuentemente
- Componentes con renderizado costoso
- Listas con muchos elementos

**¿Cómo se usa?**
```jsx
const MiComponente = React.memo(function MiComponente(props) {
  return <div>{props.valor}</div>;
});
```

**Ejemplo práctico - Lista de productos optimizada:**

<Code code={`// Componente hijo memorizado
const ProductoItem = React.memo(function ProductoItem({ producto, onAgregar }) {
  console.log('Renderizando:', producto.nombre);
  
  return (
    <div className="producto">
      <h3>{producto.nombre}</h3>
      <p>\${producto.precio}</p>
      <button onClick={() => onAgregar(producto.id)}>
        Agregar al carrito
      </button>
    </div>
  );
});

function ListaProductos({ productos }) {
  const [carrito, setCarrito] = useState([]);
  
  const agregarAlCarrito = useCallback((id) => {
    setCarrito(prev => [...prev, id]);
  }, []);
  
  return (
    <div>
      <p>Items en carrito: {carrito.length}</p>
      {productos.map(producto => (
        <ProductoItem 
          key={producto.id}
          producto={producto}
          onAgregar={agregarAlCarrito}
        />
      ))}
    </div>
  );
}`} lang="jsx" />

### 14. ¿Qué es useReducer?

**¿Qué es?**
Hook alternativo a useState para manejar estado complejo con múltiples transiciones.

**¿Para qué sirve?**
Organizar la lógica de actualización de estado en un solo lugar (el reducer), especialmente cuando:
- El estado tiene múltiples valores relacionados
- Las actualizaciones dependen del estado anterior
- La lógica es compleja

**¿Cómo se usa?**
```jsx
const [state, dispatch] = useReducer(reducer, estadoInicial);
// dispatch({ type: 'ACCION', payload: datos })
```

**Ejemplo práctico - Formulario de registro:**

<Code code={`const initialState = {
  nombre: '',
  email: '',
  password: '',
  errors: {},
  isSubmitting: false
};

function formReducer(state, action) {
  switch (action.type) {
    case 'SET_FIELD':
      return { 
        ...state, 
        [action.field]: action.value,
        errors: { ...state.errors, [action.field]: '' }
      };
    case 'SET_ERROR':
      return { 
        ...state, 
        errors: { ...state.errors, [action.field]: action.error }
      };
    case 'SUBMIT_START':
      return { ...state, isSubmitting: true };
    case 'SUBMIT_SUCCESS':
      return initialState;
    case 'SUBMIT_ERROR':
      return { ...state, isSubmitting: false };
    default:
      return state;
  }
}

function FormularioRegistro() {
  const [state, dispatch] = useReducer(formReducer, initialState);
  
  const handleChange = (e) => {
    dispatch({ 
      type: 'SET_FIELD', 
      field: e.target.name, 
      value: e.target.value 
    });
  };
  
  const handleSubmit = async (e) => {
    e.preventDefault();
    dispatch({ type: 'SUBMIT_START' });
    
    try {
      await registrarUsuario(state);
      dispatch({ type: 'SUBMIT_SUCCESS' });
    } catch (error) {
      dispatch({ type: 'SUBMIT_ERROR' });
    }
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input 
        name="nombre" 
        value={state.nombre} 
        onChange={handleChange} 
      />
      {state.errors.nombre && <span>{state.errors.nombre}</span>}
      
      <input 
        name="email" 
        value={state.email} 
        onChange={handleChange} 
      />
      
      <button disabled={state.isSubmitting}>
        {state.isSubmitting ? 'Enviando...' : 'Registrar'}
      </button>
    </form>
  );
}`} lang="jsx" />

### 15. ¿Cuándo usar useState vs useReducer?

**Regla simple:**
- **useState**: Para valores simples e independientes (un contador, un string, un boolean)
- **useReducer**: Para estado complejo con múltiples valores relacionados

**Comparación:**

| useState | useReducer |
|----------|------------|
| `setCount(count + 1)` | `dispatch({ type: 'INCREMENT' })` |
| Estado simple | Estado complejo |
| Pocas transiciones | Muchas transiciones |
| Lógica simple | Lógica compleja |
| Valores independientes | Valores relacionados |

**Ejemplo - Cuándo migrar de useState a useReducer:**

<Code code={`// Si tienes MUCHOS useState relacionados...
const [nombre, setNombre] = useState('');
const [email, setEmail] = useState('');
const [loading, setLoading] = useState(false);
const [error, setError] = useState(null);
const [success, setSuccess] = useState(false);

// ...considera usar useReducer
const [state, dispatch] = useReducer(formReducer, {
  nombre: '',
  email: '',
  loading: false,
  error: null,
  success: false
});`} lang="jsx" />

## Ciclo de Vida y Efectos (16-25)

### 16. ¿Cuál es el ciclo de vida de un componente?

**¿Qué es?** Las fases por las que pasa un componente: Montaje (se crea), Actualización (cambia), Desmontaje (se elimina).

**¿Cómo se maneja?** Con `useEffect` y su array de dependencias.

<Code code={`// Montaje
useEffect(() => {
  console.log('componentDidMount');
}, []);

// Actualización
useEffect(() => {
  console.log('componentDidUpdate');
}, [dependency]);

// Desmontaje
useEffect(() => {
  return () => {
    console.log('componentWillUnmount');
  };
}, []);`} lang="jsx" />

### 17. ¿Por qué useEffect se ejecuta dos veces en desarrollo?

**¿Qué pasa?** React 18 con StrictMode monta, desmonta y vuelve a montar componentes intencionalmente.

**¿Por qué?** Para detectar bugs en efectos que no limpian correctamente. Solo ocurre en desarrollo, NO en producción.

### 18. ¿Cómo hacer fetch de datos correctamente?

**Consideraciones:** Manejar loading, error, y cancelar requests si el componente se desmonta.

<Code code={`function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  
  useEffect(() => {
    let cancelled = false;
    
    async function fetchData() {
      try {
        setLoading(true);
        const response = await fetch(url);
        const json = await response.json();
        
        if (!cancelled) {
          setData(json);
        }
      } catch (err) {
        if (!cancelled) {
          setError(err);
        }
      } finally {
        if (!cancelled) {
          setLoading(false);
        }
      }
    }
    
    fetchData();
    
    return () => {
      cancelled = true;
    };
  }, [url]);
  
  return { data, loading, error };
}`} lang="jsx" />

### 19. ¿Qué es useLayoutEffect?

**¿Qué es?** Versión síncrona de useEffect que se ejecuta ANTES de que el navegador pinte.

**¿Para qué sirve?** Mediciones del DOM o cambios visuales que deben ocurrir antes del paint (evitar parpadeos).

<Code code={`// useEffect - asíncrono, después del paint
useEffect(() => {
  // Puede causar parpadeo visual
}, []);

// useLayoutEffect - síncrono, antes del paint
useLayoutEffect(() => {
  // Para mediciones del DOM
  const height = ref.current.offsetHeight;
}, []);`} lang="jsx" />

### 20. ¿Cómo evitar memory leaks en useEffect?

**¿Qué es?** Cuando un componente desmontado intenta actualizar su estado.

**¿Cómo prevenirlo?** SIEMPRE retornar una función de limpieza que cancele operaciones pendientes.

<Code code={`useEffect(() => {
  // 1. Cancelar fetch
  const controller = new AbortController();
  fetch(url, { signal: controller.signal });
  
  // 2. Limpiar subscripciones
  const subscription = eventEmitter.subscribe(handler);
  
  // 3. Limpiar timers
  const timerId = setInterval(() => {}, 1000);
  
  return () => {
    controller.abort();
    subscription.unsubscribe();
    clearInterval(timerId);
  };
}, []);`} lang="jsx" />

### 21. ¿Qué son los custom hooks?

**¿Qué es?** Funciones que empiezan con `use` y pueden usar otros hooks.

**¿Para qué sirve?** Extraer y reutilizar lógica con estado entre componentes.

<Code code={`// Custom hook - reutiliza lógica con estado
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });
  
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
  
  return [value, setValue];
}

// Uso
function App() {
  const [theme, setTheme] = useLocalStorage('theme', 'dark');
}`} lang="jsx" />

### 22. ¿Cuáles son las reglas de los Hooks?

**¿Qué son?** Reglas obligatorias que React necesita para que los hooks funcionen correctamente.

**¿Por qué existen?** React rastrea los hooks por el ORDEN en que se llaman. Si el orden cambia entre renders, React se confunde.

1. **Solo en el nivel superior** - No dentro de loops, condiciones o funciones anidadas
2. **Solo en componentes de React** - O en custom hooks
3. **Mismo orden siempre** - React depende del orden de llamada

<Code code={`// MAL
if (condition) {
  useEffect(() => {});
}

// BIEN
useEffect(() => {
  if (condition) {
    // lógica
  }
});`} lang="jsx" />

### 23. ¿Qué es useId?

**¿Qué es?** Hook que genera IDs únicos y estables, especialmente útil para Server-Side Rendering (SSR).

**¿Para qué sirve?** Conectar labels con inputs de forma accesible, sin conflictos de IDs entre servidor y cliente.

<Code code={`function FormField() {
  const id = useId();
  
  return (
    <>
      <label htmlFor={id}>Email</label>
      <input id={id} type="email" />
    </>
  );
}
// Genera IDs únicos y estables para SSR`} lang="jsx" />

### 24. ¿Qué es useTransition?

**¿Qué es?** Hook que permite marcar actualizaciones de estado como "no urgentes" para mantener la UI responsiva.

**¿Para qué sirve?** Evitar que operaciones costosas (filtrar listas grandes) bloqueen la interacción del usuario.

<Code code={`function App() {
  const [isPending, startTransition] = useTransition();
  const [query, setQuery] = useState('');
  
  const handleChange = (e) => {
    // Actualización urgente
    setQuery(e.target.value);
    
    // Actualización no urgente
    startTransition(() => {
      setFilteredItems(filterItems(e.target.value));
    });
  };
  
  return (
    <>
      <input onChange={handleChange} />
      {isPending && <Spinner />}
    </>
  );
}`} lang="jsx" />

### 25. ¿Qué es useDeferredValue?

**¿Qué es?** Hook que retrasa la actualización de un valor hasta que el navegador tenga tiempo libre.

**¿Para qué sirve?** Similar a useTransition, pero para valores que vienen de props en vez de estado local.

<Code code={`function SearchResults({ query }) {
  // Retrasa la actualización de query
  const deferredQuery = useDeferredValue(query);
  
  // Muestra resultados con el valor diferido
  const results = useMemo(
    () => searchItems(deferredQuery),
    [deferredQuery]
  );
  
  return <List items={results} />;
}`} lang="jsx" />

## Patrones y Arquitectura (26-35)

### 26. ¿Qué es el patrón Container/Presentational?

**¿Qué es?** Patrón que separa componentes en dos tipos: los que manejan lógica (Container) y los que solo renderizan UI (Presentational).

**¿Para qué sirve?** Mejor organización, reutilización de UI, y testing más fácil.

<Code code={`// Container - lógica y estado
function UserListContainer() {
  const [users, setUsers] = useState([]);
  
  useEffect(() => {
    fetchUsers().then(setUsers);
  }, []);
  
  return <UserList users={users} />;
}

// Presentational - solo UI
function UserList({ users }) {
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}`} lang="jsx" />

### 27. ¿Qué es el patrón Compound Components?

**¿Qué es?** Patrón donde varios componentes trabajan juntos compartiendo estado implícito via Context.

**¿Para qué sirve?** Crear APIs limpias y flexibles (como `<select>` y `<option>` en HTML).

<Code code={`// API limpia para componentes relacionados
function Tabs({ children, defaultTab }) {
  const [activeTab, setActiveTab] = useState(defaultTab);
  
  return (
    <TabsContext.Provider value={{ activeTab, setActiveTab }}>
      {children}
    </TabsContext.Provider>
  );
}

Tabs.List = function TabList({ children }) {
  return <div className="tab-list">{children}</div>;
};

Tabs.Tab = function Tab({ id, children }) {
  const { activeTab, setActiveTab } = useContext(TabsContext);
  return (
    <button 
      className={activeTab === id ? 'active' : ''}
      onClick={() => setActiveTab(id)}
    >
      {children}
    </button>
  );
};

// Uso
<Tabs defaultTab="tab1">
  <Tabs.List>
    <Tabs.Tab id="tab1">Tab 1</Tabs.Tab>
    <Tabs.Tab id="tab2">Tab 2</Tabs.Tab>
  </Tabs.List>
</Tabs>`} lang="jsx" />

### 28. ¿Qué es el patrón Render Props?

**¿Qué es?** Patrón donde un componente recibe una función como prop que retorna JSX.

**¿Para qué sirve?** Compartir lógica entre componentes de forma flexible. Hoy se prefieren custom hooks.

<Code code={`// Componente que comparte lógica via render prop
function Mouse({ render }) {
  const [position, setPosition] = useState({ x: 0, y: 0 });
  
  useEffect(() => {
    const handleMove = (e) => {
      setPosition({ x: e.clientX, y: e.clientY });
    };
    window.addEventListener('mousemove', handleMove);
    return () => window.removeEventListener('mousemove', handleMove);
  }, []);
  
  return render(position);
}

// Uso
<Mouse render={({ x, y }) => (
  <p>Posición: {x}, {y}</p>
)} />`} lang="jsx" />

### 29. ¿Qué son los Higher-Order Components (HOC)?

**¿Qué es?** Función que recibe un componente y retorna un nuevo componente con funcionalidad adicional.

**¿Para qué sirve?** Reutilizar lógica (autenticación, logging, etc.). Hoy se prefieren custom hooks.

<Code code={`// HOC - función que recibe componente y retorna componente mejorado
function withAuth(WrappedComponent) {
  return function AuthenticatedComponent(props) {
    const { user } = useAuth();
    
    if (!user) {
      return <Navigate to="/login" />;
    }
    
    return <WrappedComponent {...props} user={user} />;
  };
}

// Uso
const ProtectedDashboard = withAuth(Dashboard);`} lang="jsx" />

### 30. ¿Cuándo usar HOC vs Custom Hooks?

**¿Cuál elegir?** Custom Hooks son la opción moderna y preferida. HOCs son legacy pero aún útiles en casos específicos.

| HOC | Custom Hooks |
|-----|--------------|
| Modifica el árbol de componentes | Solo comparte lógica |
| Puede interceptar props | Más flexible |
| Más difícil de tipar | Mejor TypeScript |
| Legacy | Moderno (preferido) |

### 31. ¿Qué es forwardRef?

**¿Qué es?** Función que permite a un componente recibir una ref y pasarla a un elemento hijo.

**¿Para qué sirve?** Acceder al DOM de un componente hijo desde el padre (ej: enfocar un input custom).

<Code code={`// Permite pasar ref a componentes hijos
const Input = forwardRef(function Input(props, ref) {
  return <input ref={ref} {...props} />;
});

// Uso
function Form() {
  const inputRef = useRef(null);
  
  const focusInput = () => {
    inputRef.current.focus();
  };
  
  return <Input ref={inputRef} />;
}`} lang="jsx" />

### 32. ¿Qué es useImperativeHandle?

**¿Qué es?** Hook que personaliza qué métodos expone un componente cuando usa forwardRef.

**¿Para qué sirve?** Exponer solo métodos específicos al padre, en vez de todo el elemento DOM.

<Code code={`const FancyInput = forwardRef(function FancyInput(props, ref) {
  const inputRef = useRef(null);
  
  // Expone solo métodos específicos
  useImperativeHandle(ref, () => ({
    focus: () => inputRef.current.focus(),
    clear: () => inputRef.current.value = ''
  }));
  
  return <input ref={inputRef} />;
});

// Uso
function Parent() {
  const ref = useRef(null);
  
  return (
    <>
      <FancyInput ref={ref} />
      <button onClick={() => ref.current.focus()}>Focus</button>
      <button onClick={() => ref.current.clear()}>Clear</button>
    </>
  );
}`} lang="jsx" />

### 33. ¿Qué es Suspense?

**¿Qué es?** Componente que muestra un fallback mientras sus hijos están "suspendidos" (cargando).

**¿Para qué sirve?** Mostrar loading states de forma declarativa para lazy loading y data fetching.

<Code code={`// Suspense para lazy loading
const LazyComponent = lazy(() => import('./HeavyComponent'));

function App() {
  return (
    <Suspense fallback={<Loading />}>
      <LazyComponent />
    </Suspense>
  );
}

// Suspense para data fetching (React 18+)
function Posts() {
  return (
    <Suspense fallback={<Skeleton />}>
      <PostList />
    </Suspense>
  );
}`} lang="jsx" />

### 34. ¿Qué son los Error Boundaries?

**¿Qué es?** Componentes de clase que capturan errores de JavaScript en su árbol de hijos.

**¿Para qué sirve?** Evitar que un error en un componente rompa toda la app. Mostrar UI de error amigable.

<Code code={`class ErrorBoundary extends React.Component {
  state = { hasError: false };
  
  static getDerivedStateFromError(error) {
    return { hasError: true };
  }
  
  componentDidCatch(error, errorInfo) {
    logErrorToService(error, errorInfo);
  }
  
  render() {
    if (this.state.hasError) {
      return <h1>Algo salió mal.</h1>;
    }
    return this.props.children;
  }
}

// Uso
<ErrorBoundary>
  <MiComponente />
</ErrorBoundary>`} lang="jsx" />

### 35. ¿Qué es React.lazy?

**¿Qué es?** Función que permite cargar componentes de forma diferida (lazy loading).

**¿Para qué sirve?** Code splitting - dividir el bundle en partes más pequeñas que se cargan bajo demanda.

<Code code={`// Code splitting - carga componentes bajo demanda
const Dashboard = lazy(() => import('./Dashboard'));
const Settings = lazy(() => import('./Settings'));

function App() {
  return (
    <Suspense fallback={<Loading />}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/settings" element={<Settings />} />
      </Routes>
    </Suspense>
  );
}`} lang="jsx" />

## React Router y Navegación (36-40)

### 36. ¿Cómo configurar React Router?

**¿Qué es?** Librería estándar para manejar navegación en React SPAs.

**Componentes principales:** `BrowserRouter` (envuelve la app), `Routes` (contenedor), `Route` (define rutas), `Link` (navegación).

<Code code={`import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Inicio</Link>
        <Link to="/about">Acerca</Link>
      </nav>
      
      <Routes>
        <Route path="/" element={<Home />} />

        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
}`} lang="jsx" />

### 37. ¿Cómo obtener parámetros de la URL?

**Hooks disponibles:**
- `useParams()` - Parámetros de ruta (`:id`)
- `useSearchParams()` - Query params (`?page=1`)

<Code code={`import { useParams, useSearchParams } from 'react-router-dom';

function User() {
  // Parámetros de ruta: /users/:id
  const { id } = useParams();
  
  // Query params: /users?page=1&sort=name
  const [searchParams, setSearchParams] = useSearchParams();
  const page = searchParams.get('page');
  
  return <div>Usuario {id}, página {page}</div>;
}`} lang="jsx" />

### 38. ¿Cómo hacer navegación programática?

**¿Qué es?** Navegar desde código JavaScript en vez de clicks del usuario.

**¿Cómo?** Usar el hook `useNavigate()` que retorna una función para navegar.

<Code code={`import { useNavigate } from 'react-router-dom';

function Login() {
  const navigate = useNavigate();
  
  const handleLogin = async () => {
    await login();
    navigate('/dashboard');
    // o navigate(-1) para ir atrás
    // o navigate('/dashboard', { replace: true })
  };
}`} lang="jsx" />

### 39. ¿Cómo proteger rutas?

**¿Qué es?** Restringir acceso a ciertas rutas solo a usuarios autenticados.

**¿Cómo?** Crear un componente wrapper que verifica autenticación y redirige si no hay usuario.

<Code code={`function ProtectedRoute({ children }) {
  const { user } = useAuth();
  const location = useLocation();
  
  if (!user) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }
  
  return children;
}

// Uso

  path="/dashboard" 
  element={

      <Dashboard />
    </ProtectedRoute>
  } 
/>`} lang="jsx" />

## Testing (41-45)

### 41. ¿Cómo testear componentes con React Testing Library?

**¿Qué es?** Librería que testea componentes como los usaría un usuario real.

**Filosofía:** Buscar elementos por rol, texto o label (no por clases o IDs).

<Code code={`import { render, screen, fireEvent } from '@testing-library/react';

test('incrementa contador al hacer clic', () => {
  render(<Counter />);
  
  const button = screen.getByRole('button', { name: /incrementar/i });
  const count = screen.getByText(/contador: 0/i);
  
  fireEvent.click(button);
  
  expect(screen.getByText(/contador: 1/i)).toBeInTheDocument();
});`} lang="jsx" />

### 42. ¿Cómo testear hooks personalizados?

**¿Cómo?** Usar `renderHook` de Testing Library. Envolver cambios de estado en `act()`.

<Code code={`import { renderHook, act } from '@testing-library/react';

test('useCounter incrementa valor', () => {
  const { result } = renderHook(() => useCounter(0));
  
  expect(result.current.count).toBe(0);
  
  act(() => {
    result.current.increment();
  });
  
  expect(result.current.count).toBe(1);
});`} lang="jsx" />

### 43. ¿Cómo mockear fetch en tests?

**¿Por qué?** Evitar llamadas reales a APIs en tests. Controlar las respuestas.

**¿Cómo?** Usar `jest.fn()` para reemplazar `fetch` global.

<Code code={`beforeEach(() => {
  global.fetch = jest.fn(() =>
    Promise.resolve({
      json: () => Promise.resolve({ data: 'test' })
    })
  );
});

afterEach(() => {
  jest.restoreAllMocks();
});

test('carga datos correctamente', async () => {
  render(<DataComponent />);
  
  await waitFor(() => {
    expect(screen.getByText('test')).toBeInTheDocument();
  });
});`} lang="jsx" />

### 44. ¿Qué es snapshot testing?

**¿Qué es?** Guardar el output renderizado y compararlo en futuros tests.

**¿Para qué sirve?** Detectar cambios inesperados en la UI. Útil pero puede dar falsos positivos.

<Code code={`test('renderiza correctamente', () => {
  const { asFragment } = render(<Button>Click me</Button>);
  expect(asFragment()).toMatchSnapshot();

### 47. ¿Qué causa re-renders innecesarios?
